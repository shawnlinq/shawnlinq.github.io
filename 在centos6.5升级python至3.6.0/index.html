在Centos6.5下升级python至3.6.0

Python最新版已经不是3.6.0了, 权当做参考

1，配置系统环境

1.1，更新centos系统

$ yum -y update

1.2，安装开发工具

$ yum groupinstall -y developement

1.3，安装python3解码支持包

$ yum install -y zlib-devel openssl-devel sqlite-devel bzip2-devel

2，更新Python版本

centos6.5默认使用的python版本为2.6.6（需要自己验证一下），现升级到python3.6.0

2.1，验证python版本

$ python -V

python 2.6.6

2.2，下载python3.6.0安装包

$ wget http://www.python.org/ftp/python/3.6.0/Python-3.6.0.tar.xz

2.3，解压编译python安装包

2.3.1，解压

$ xz -d Python-3.6.0.tar.xz  
$ tar -xvf Python-3.6.0.tar

2.3.2，编译

$ cd Python-3.6.0  
$ ./configure --prefix=/usr/local

如果出现编译错误可能是因为gcc gcc-c++版本太低或者未安装，使用代码 

$ yum -y install gcc gcc-c++

进行安装，然后重新编译 

$ ./configure

2.4，执行安装

$ make && make altinstall

2.5，建立软连接

$ mv /usr/bin/python /usr/bin/python2.6.6    ##你的python原本可能不同
$ ln -s /usr/local/bin/python3.6 /usr/bin/python

重新验证python版本， 

$ python -V

2.6，yum指令会报错，将其重新指向旧版本的python

$ vi /usr/bin/yum

将文件的头部#！/usr/bin/python改为#！/usr/bin/python2.6.6

用Inster键来插入修改，修改完后使用ESC键，输入：wq来保存退出

3，安装pip

3.1，下载pip安装包

$ wget https://pypi.python.org/packages/source/p/pip/pip-1.3.1.tar.gz --no-check-certificate

由于下载pip是基于https协议的，故需要在wget的url后面加上--no-check-certificate，否则不能下载

3.2，解压安装pip

$ chmod +x pip-1.3.1.tar.gz
$ tar xzvf pip-1.3.1.tar.gz
$ cd pip-1.3.1
$ python setup.py install

3.3，用pip安装virtualenv验证安装正确

$ pip install virtualenv

至此，centos6.5的python版本已经升级为3.6.0
